# Get the payment details

To get the payment status you will make a <u>[`GET /api/v1/payment/status`](../../../api/gateway/check-transaction-status)</u> <br></br>
You will retrieve the payment status and if it's declinced you will receive the reason .

---

## Types of payment status

the payment status is a way for the gateway to tell you in what state your transaction in , and based on that you will make the appropriate action.

By default you will recieve a `callback_notification` but in case if you want to make a follow up request to check for the payment status, you can make a <u>[`GET /api/v1/payment/status`](../../../api/gateway/check-transaction-status)</u>.

# Status

payment status will be one of these

- prepare
- settled
- pending
- 3ds
- redirect
- decline
- refund
- reversal
- chargeback

> in case the transaction is not successful you receive a `reason` pramater in the response

## Sample request

to check the payment status you will have to pass the `payment_id` so make sure to save it , once you recieve it from the `callback_notification`.

```json
{
	"merchant_key": " xxxxx-xxxxx-xxxxx",
	"payment_id": "dc66cdd8-d702-11ea-9a2f-0242c0a87002",
	"hash": "xxxxxxxxxxx"
}
```
